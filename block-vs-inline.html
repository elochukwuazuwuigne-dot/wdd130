Exercise: Block vs. Inline Elements

Some basic styling has been applied in order to illustrate the elements structure and boundaries.

Task 1: Swap Block and Inline Elements

In the HTML of this document, change this div element to a span element.

What happens to the layout? Does it still take up the full width?

Answer: The span element is inline, so it no longer takes up the full width. It only occupies the width of its content, unlike a div which is block and stretches across the container.

Task 2: Nesting Inline in Block

A paragraph <p> element (block) with a <span> element;

Add another inline element like <em> or <strong> inside the paragraph. Observe how it flows within the line.

This is a span element and here is an emphasized text and a strong text. Inline elements flow within the line of the paragraph and do not start on a new line.

Task 3: Nesting Block in Inline

Try placing a <div> inside a <span> element.

What error or layout issue do you see? Why does this happen?

Answer: Placing a block element like <div> inside an inline element like <span> is invalid HTML. Browsers may automatically close the <span> before the <div> or move it outside. Inline elements cannot contain block-level elements.
